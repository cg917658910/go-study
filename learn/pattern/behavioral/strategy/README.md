# Strategy Pattern (策略模式)

## 定义
策略模式定义一系列算法,把它们一个个封装起来,并且使它们可相互替换。

## 目的
- 封装算法
- 算法可替换
- 避免条件语句

## 使用场景
- 排序算法选择
- 压缩算法选择
- 支付方式选择
- 数据验证策略

## Go 特有实现
使用接口或函数类型:

```go
type Strategy interface {
    Execute(a, b int) int
}

// 或使用函数类型
type StrategyFunc func(a, b int) int
```

## 优点
1. **开闭原则** - 易于扩展新算法
2. **避免条件语句** - 消除大量 if-else
3. **算法独立** - 算法可独立变化

## 缺点
1. **类增多** - 每个策略需要一个类
2. **客户端必须了解策略** - 需要知道各策略的差异
