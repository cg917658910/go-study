# Chain of Responsibility Pattern (责任链模式)

## 定义
责任链模式使多个对象都有机会处理请求,从而避免请求的发送者和接收者之间的耦合关系。

## 目的
- 解耦发送者和接收者
- 多个对象可以处理同一个请求
- 动态添加处理者

## 使用场景
- HTTP 中间件链
- 事件处理链
- 审批流程
- 日志处理(按级别过滤)

## 优点
1. **降低耦合度** - 发送者和接收者解耦
2. **灵活性** - 可动态添加或删除处理者
3. **职责分配** - 每个处理者专注于自己的职责

## 缺点
1. **性能影响** - 请求可能经过很多处理者
2. **调试困难** - 难以观察运行时特征

## Go 特有实现
使用接口和链表结构,或者函数链。
