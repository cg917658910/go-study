# Abstract Factory Pattern (抽象工厂模式)

## 定义
抽象工厂模式提供一个接口,用于创建相关或依赖对象的家族,而不需要指定它们的具体类。

## 目的
- 创建产品家族
- 确保产品一致性
- 隔离具体类的生成

## 使用场景
- UI 组件库(不同主题的按钮、输入框、对话框)
- 跨平台应用(Windows、Linux、MacOS 组件)
- 数据库访问层(不同数据库的连接、命令、事务对象)

## 优点
1. **产品一致性** - 保证产品家族的一致性
2. **易于切换** - 更换产品家族很容易
3. **单一职责** - 产品创建集中管理

## 缺点
1. **扩展困难** - 添加新产品需要修改接口
2. **复杂度高** - 引入大量类和接口
