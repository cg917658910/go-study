# Composite Pattern (组合模式)

## 定义
组合模式将对象组合成树形结构以表示"部分-整体"的层次结构,使客户端对单个对象和组合对象的使用具有一致性。

## 目的
- 表示部分-整体层次
- 统一对待单个对象和组合对象
- 简化客户端代码

## 使用场景
- 文件系统(文件和文件夹)
- UI 组件树(容器和控件)
- 组织架构(部门和员工)
- 菜单系统(菜单和菜单项)

## 优点
1. **统一接口** - 一致对待叶子和组合
2. **易于扩展** - 容易添加新组件
3. **简化客户端** - 客户端代码简单

## 缺点
1. **设计过度泛化** - 可能难以限制组件类型
2. **类型检查困难** - 运行时类型检查复杂
