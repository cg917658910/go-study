# Flyweight Pattern (享元模式)

## 定义
享元模式运用共享技术有效地支持大量细粒度的对象,减少内存使用。

## 目的
- 减少内存占用
- 提高性能
- 共享相同状态的对象
- 分离内部状态和外部状态

## 使用场景
- 系统有大量相似对象
- 对象的大部分状态可以外部化
- 需要缓冲池的场景

## 优点
1. **减少内存占用** - 共享对象
2. **提高性能** - 减少对象创建

## 缺点
1. **增加系统复杂度** - 需要分离内外部状态
2. **运行时间换空间** - 查找共享对象需要时间

## Go 特有实现
使用 map 或 sync.Pool 实现对象共享。
