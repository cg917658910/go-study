# Bridge Pattern (桥接模式)

## 定义
桥接模式将抽象部分与它的实现部分分离,使它们都可以独立地变化。

## 目的
- 分离抽象与实现
- 支持多维度变化
- 避免类爆炸
- 提高可扩展性

## 使用场景
- 不希望在抽象和实现之间有固定绑定
- 抽象和实现都应该可以通过子类扩展
- 对抽象的实现部分的修改不影响客户端
- 需要在多个对象间共享实现

## 优点
1. **分离抽象和实现** - 提高可扩展性
2. **提高可扩充性** - 独立扩展抽象和实现
3. **实现细节对客户透明** - 隐藏实现细节

## 缺点
1. **增加系统理解难度** - 需要正确识别两个独立变化的维度
2. **增加系统复杂度** - 引入更多的类

## Go 特有实现
使用组合和接口实现抽象与实现的分离。
